## Принцип работы

Один компьютер **отправляет** файлы, отображая поток QR-кодов на экране.  
Другой компьютер **принимает**, направляя веб-камеру на экран отправителя.

Данные сжимаются, разбиваются на кадры и защищаются **кодами Рида-Соломона (FEC)** — даже если камера пропустит часть кадров, файл будет полностью восстановлен.

## Возможности

- **Полностью офлайн** — не нужны Wi-Fi, Bluetooth или интернет
- **GUI в один клик** — тёмный терминальный интерфейс, перетаскивание файлов
- **FEC-восстановление** — устойчивость к пропущенным кадрам через коды Рида-Соломона над GF(256)
- **Межблочное чередование** — последовательные потери кадров распределяются между блоками
- **Настраиваемый FPS** — подстройка скорости под возможности камеры
- **Адресная повторная отправка** — клик по блоку для повторной отправки недостающих кадров
- **Многоязычный интерфейс** — English, 中文, Русский, 日本語

## Быстрый старт (из исходников)

### Требования

- Python 3.9+
- Веб-камера на принимающем компьютере

### Установка

```bash
git clone https://github.com/patrickconstfiv/AirQRT.git
cd AirQRT
pip install -r requirements.txt
```

### Запуск

```bash
python app.py
```

Откроется GUI с двумя вкладками: **ОТПР** и **ПРИЁМ**.

#### Отправка

1. Перейдите на вкладку **ОТПР**
2. Нажмите **Добавить** (или перетащите файлы)
3. При необходимости настройте FPS
4. Нажмите **[ НАЧАТЬ ОТПРАВКУ ]**

#### Приём

1. Перейдите на вкладку **ПРИЁМ**
2. Направьте камеру на экран отправителя
3. Нажмите **[ НАЧАТЬ ПРИЁМ ]**
4. После завершения файлы сохраняются в `received_files/`

## Сборка исполняемого файла

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name "AirQRT" --icon=icon.ico --hidden-import=windnd app.py
```

`.exe` появится в папке `dist/`.

## FEC-восстановление ошибок

Система использует блочный FEC на основе кодов Рида-Соломона:

| Параметр | Значение по умолчанию |
|----------|----------------------|
| Кадров данных на блок | 50 |
| Доля избыточности | 30% |
| Уровень коррекции QR | M (15%) |

**Пример:** Блок из 50 кадров данных генерирует 15 кадров чётности (всего 65).  
Если считаны **любые** 50 из 65 кадров — весь блок восстанавливается полностью.

## Производительность

| Размер файла | Примерное время |
|-------------|-----------------|
| 10 КБ | ~5 сек |
| 50 КБ | ~15 сек |
| 100 КБ | ~30 сек |
| 500 КБ | ~2,5 мин |

Подходит для файлов до 500 КБ. Крупные файлы рекомендуется сжать перед отправкой.

## Настройка параметров

Отредактируйте `sender.py`:

```python
CHUNK_SIZE = 300            # байт на шард данных (меньше = легче считать)
FRAME_FPS = 20              # FPS по умолчанию
FEC_DATA_SHARDS = 50        # кадров данных на блок FEC
FEC_REDUNDANCY_RATIO = 0.30 # 30% избыточности
QR_ERROR_LEVEL = 'M'        # L / M / Q / H
```

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Камера не открывается | Измените `CAMERA_INDEX` в `receiver_camera.py` на `1` или `2` |
| Низкая скорость считывания | Уменьшите `CHUNK_SIZE`, снизьте FPS или увеличьте `FEC_REDUNDANCY_RATIO` |
| Большой файл передаётся медленно | Сначала сожмите файл (zip/7z), затем отправьте архив |

## Структура проекта

```
├── sender.py            # Сжатие файлов, кодирование QR, FEC-кадрирование

├── icon.png             # Иконка приложения
├── build_exe.bat        # Скрипт сборки EXE
├── requirements.txt     # Зависимости Python
└── received_files/      # Каталог полученных файлов
```
